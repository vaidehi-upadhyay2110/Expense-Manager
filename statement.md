Problem Statement : 
Individuals lack an immediate, reliable, and secure system to log, categorize, and monitor their daily expenditures, leading to poor financial visibility and control.

Scope of the Project :
Focuses on providing a full CRUD cycle (Create, Read, Update, Delete) for expense data with real-time synchronization using Firestore and user-specific data isolation. 
Excludes: Reporting, budgeting, or advanced visualization features.

Target Users :
Individuals, freelancers, or small households who need a quick, secure, and persistent tool to track personal or small-scale financial transactions in real-time.

High-Level Features :
1. Real-Time CRUD: Instantaneous creation, modification, and deletion of expense records.
2. Categorization: Tagging expenses for organized monitoring.
3. Live Balance Calculation: Displays updated totals instantly. 
4.  Persistent & Secure Storage: Data is saved to Firestore.
